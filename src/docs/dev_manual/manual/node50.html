10.3.3 Creating an Archive

Suppose you have run a simulation, with the run-name ``xyzzy'', and have all the input and output files in some directory. To archive the run do the following :

  1. As the user who ran the simulation :
    1. Go to the directory with the ``xyzzy'' data.
    2. Create a README file with information about :
      • what the simulation is.
      • who did the simulation.
      • when the simulation was done.
      • where it was done and with what version of the code.
      • any other information needed for someone to recreate the data.
    3. Make sure all data is readable by user parflow.
  2. As user parflow :
    1. Create parflow/data/input/xyzzy.
    2. Give parflow/data/input/xyzzy permissions consistent with the level of security needed for the input file directory § 10.2.
    3. Copy to parflow/data/input/xyzzy :
      • all xyzzy.in.* files used in the simulation.
      • all geo-unit input files used in the simulation.
      • the README file.
    4. Give all files in parflow/data/input/xyzzy permissions consistent with the level of security needed for the input files themselves (including any special executables) § 10.2.
    5. Create parflow/data/output/xyzzy.
    6. Give parflow/data/output/xyzzy permissions consistent with the level of security needed for the output file directory § 10.2.
    7. Copy to parflow/data/output/xyzzy :
      • all xyzzy.out.* files from the simulation.
    8. Give all files in parflow/data/output/xyzzy permissions consistent with the level of security needed for the output files themselves (including any special executables) § 10.2.
  3. As the user who ran the simulation :
    1. Go to the directory parflow/data and do the following :
      1. Checkout (and lock) the file INFO.html.
      2. Edit the file INFO.html adding lines for run ``xyzzy'' similar to the other entries (making sure to note that the data is not backed up at this time) and add a link to file input/xyzzy/README (your README file).
      3. Checkin the file INFO.html.

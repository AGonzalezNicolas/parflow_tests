2.2 Building Code

All code and documentation are ``built'' with a build script. When writing new build scripts, the following build command conventions should be followed:

  • build all -- builds things
  • build install -- builds and installs exe's and support files. Installation usually involves copying of files into the $PARFLOW_DIR and $PARFLOW_HELP directories. However, in some cases, files are installed elsewhere.
  • build clean -- clean object files (and some other temp stuff)
  • build veryclean -- cleans object files and exe's within the $PARFLOW_SRC directory (i.e., does not remove ``installed'' files).

This build technology contains some ``smarts''. For example, it detects when you change platforms (say you log into the SGI) and will delete object files and remake everything.

Notes:

  • Because code is ``copied'' when installed (as opposed to using soft links, for example), when changes are made to any source, help files, input files, etc., you must do a build install to update the changes in the ``install'' directory.

  • You can safely ignore this item if you run parflow out of your home directory. Also, I believe that this only impacts the Chameleon port so you can probably safely ignore all this.

    Due to some NFS temp mount problems there are some limitations to how we can run things. The problem is attempting to get the ``virtual'' current directory rather than the ``real'' directory name. For example, if you do a pwd, you will sometimes see /home/mercury0/ssmith/blah/foo or /export/home/mercury0/ssmith/blah/foo (if you are on the machine where your home directory is), and other times you will see /tmp_mnt/home/mercury0/ssmith/blah/foo (if you are on a remote machine). This is a problem when running some of the different models since they need to see the same directory (/home/mercury0/ssmith/blah/foo is the ``correct'' name). To handle this, everything is based off of the AMPS_ROOT_DIR environment variable. By default this is set to your home directory. If the directory that you are running from is in your $HOME directory, then you are fine. If you want to run from somewhere else (say /scratch/parflow/exe) you need to either put a softlink in your home directory or explicitly set AMPS_ROOT_DIR to /scratch. You do not have to worry about this if you run out of your home directory.
